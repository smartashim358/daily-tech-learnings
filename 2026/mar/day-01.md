#  1. What Are Windows Event Logs?

##  Definition

Windows Event Logs are structured records generated by the Windows
operating system\
to document system activity, security events, and application behavior.

##  Core Concept

Every important action inside Windows leaves a trace.\
These traces are stored as `.evtx` files.

 Default Location:

    C:\Windows\System32\winevt\Logs\

##  Why Event Logs Matter

Event logs allow defenders to answer:

-   Who performed an action?
-   When did it happen?
-   From where did it originate?
-   What process was involved?
-   Was the action successful?

Without logs → No history\
With logs → Full investigative visibility

#  2. Core Log Categories

##  A. System Log

### Definition

Records operating system-level events.

### Includes

-   Service start and stop
-   Driver loading
-   Shutdown and reboot
-   Hardware errors

### Security Value

Detects: - Boot persistence - Service manipulation - Unexpected
shutdowns

------------------------------------------------------------------------

##  B. Application Log

### Definition

Records events generated by installed applications.

### Includes

-   Application crashes
-   Errors
-   Warnings

### Security Value

Helps identify: - Exploited applications - Repeated abnormal behavior

------------------------------------------------------------------------

##  C. Security Log (Most Important)

### Definition

Records authentication and security-sensitive activity.

### Includes

-   Successful logons
-   Failed logons
-   Privilege usage
-   Account changes
-   Process creation (if enabled)

### Security Value

Primary source for detecting attacks.

------------------------------------------------------------------------

#  3. Accessing Logs with PowerShell

##  List All Logs

``` powershell
Get-WinEvent -ListLog *
```

##  View Latest 10 Security Logs

``` powershell
Get-WinEvent -LogName Security -MaxEvents 10
```

##  Filter by Event ID (Example: 4624)

``` powershell
Get-WinEvent -FilterHashtable @{
    LogName='Security'
    Id=4624
} -MaxEvents 5
```

###  Concept Behind Filtering

Filtering reduces noise and focuses only on relevant events.

#  4. Critical Event IDs You Must Know

##  4624 -- Successful Logon

### Definition

A user successfully authenticated.

### Important Fields

-   Account Name
-   Logon Type
-   Source Network Address
-   Workstation Name

### Example Investigation

If login occurs at unusual time or from unknown IP → Investigate.

------------------------------------------------------------------------

##  4625 -- Failed Logon

### Definition

Authentication attempt failed.

### Detection Pattern

Multiple 4625 events → Followed by 4624 → Possible brute-force success.

------------------------------------------------------------------------

##  4688 -- Process Creation

### Definition

A new process was created.

### Important Fields

-   New Process Name
-   Creator Process Name
-   Command Line

### Example

If:

    winword.exe → powershell.exe

This may indicate macro-based attack.

------------------------------------------------------------------------

##  7045 -- Service Installed (System Log)

### Definition

A new service was installed.

### Red Flags

-   Unknown service name
-   Executable in AppData or Temp
-   Running as SYSTEM

------------------------------------------------------------------------

#  5. Structure of a Windows Event

Each event contains structured properties:

-   TimeCreated
-   Event ID
-   ProviderName
-   Level
-   Message
-   Security Identifier (SID)

##  View Full Event Structure

``` powershell
Get-WinEvent -LogName Security -MaxEvents 1 | Format-List *
```

##  Important Concept

Events are structured objects, not plain text. They can be filtered,
parsed, and correlated.

------------------------------------------------------------------------

#  6. Investigation Flow Model

## Step-by-Step Logic

1.  Detect anomaly
2.  Identify related Event ID
3.  Analyze timestamp
4.  Identify account involved
5.  Correlate with process or service activity

------------------------------------------------------------------------

##  Example Attack Chain

    Multiple 4625 (failed logons)
            ↓
    4624 (successful login)
            ↓
    4688 (process execution)
            ↓
    7045 (service installation)

### Logical Conclusion

Brute force → Access → Execution → Persistence
